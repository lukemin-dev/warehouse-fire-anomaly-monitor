# 개선 로드맵

현재 저장소는 캡스톤 시연을 위한 MVP 구현입니다. 실제 운영 수준으로 확장하려면 아래 항목을 보완합니다.

## 우선순위 높음

- 관리자 API 인증 추가
  - `/api/ai/reset`, `/api/ai/retrain`은 실제 운영 시 인증이 필요합니다.
  - API key, JWT, 관리자 계정 기반 인증 중 하나를 적용합니다.

- 서버 주소 설정 일원화
  - 현재는 `.env.example`을 기준으로 서버 주소를 주입합니다.
  - 운영 배포 시 환경별 `.env`와 배포 문서를 분리합니다.

- 회로 보호
  - 라즈베리파이와 MCP3008은 3.3V 기준입니다.
  - MQ-2 모듈의 AO 출력이 3.3V를 초과할 수 있으므로 전압 분배 또는 레벨 보호 회로를 추가합니다.

## 우선순위 중간

- 푸시 알림
  - 현재 앱은 폴링 기반으로 경고 이력을 확인합니다.
  - Expo Notifications 또는 FCM으로 즉시 알림을 추가할 수 있습니다.

- 장기 데이터 저장
  - SQLite는 시연에는 충분하지만 장기 운영에는 관리가 어렵습니다.
  - PostgreSQL, TimescaleDB, InfluxDB 같은 시계열 저장소를 검토합니다.

- 모델 평가 지표
  - 정상/이상 샘플을 분리해 precision, recall, false positive rate를 기록합니다.
  - 센서별 baseline을 더 많이 수집해 모델 안정성을 높입니다.

## 우선순위 낮음

- Docker 기반 서버 실행
- GitHub Actions로 정적 검사 자동화
- 앱 EAS Build 자동화
- 웹 대시보드 UI 분리
- 장치 등록 및 다중 창고 지원

## 면접에서 말하기 좋은 개선 포인트

캡스톤에서는 실제 화재 데이터를 대량으로 만들기 어렵기 때문에 정상 데이터 기반 이상 탐지를 선택했습니다. 운영 단계에서는 전압 보호 회로, 인증, 푸시 알림, 장기 데이터 저장, 모델 평가 지표를 보완해 안정성을 높이겠습니다.
