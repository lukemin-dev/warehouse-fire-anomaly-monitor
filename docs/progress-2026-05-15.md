## 1. MCP3008 도입 배경

### 기존 문제

기존 DO(디지털) 핀만 사용 시 센서 출력이 0 또는 1밖에 없음.

```
정상: 1, 1, 1, 1, 1
감지: 0, 0, 0, 0, 0
```

이 경우 AI(IsolationForest)가 할 수 있는 역할이 없음. if문과 차이 없음.

### 해결책

MQ-2, 불꽃 센서의 AO(아날로그 출력) 핀을 MCP3008 ADC 칩을 통해 0~1023 연속값으로 수집.

```
정상:  45,  48,  43,  50,  47
전조: 120, 180, 250, 310, 420  ← 아직 DO는 정상(1)
감지: 800, 950, 1023, 980, 900
```

AI가 전조 단계(DO는 정상인데 AO값이 올라가는 구간)를 먼저 감지 가능.

## 2. MCP3008 배선

```
MCP3008 핀      →  라즈베리파이 핀
VDD  (16)       →  3.3V (1번)
VREF (15)       →  3.3V (1번)
AGND (14)       →  GND (6번)
CLK  (13)       →  GPIO11 SCLK (23번)
DOUT (12)       →  GPIO9  MISO (21번)
DIN  (11)       →  GPIO10 MOSI (19번)
CS   (10)       →  GPIO8  CE0  (24번)
DGND (9)        →  GND (6번)

MQ-2   AO       →  MCP3008 CH0 (1번)
불꽃센서 AO     →  MCP3008 CH1 (2번)

MQ-2   DO       →  GPIO17 (기존 그대로)
불꽃센서 DO     →  GPIO27 (기존 그대로)
```

## 3. SPI 활성화

### 방법 1: raspi-config (GUI)
```bash

```

## 4. MCP3008 테스트 코드

```python
import spidev
import time

spi = spidev.SpiDev()
spi.open(0, 0)
spi.max_speed_hz = 1350000

def read_channel(channel):
    adc = spi.xfer2([1, (8 + channel) << 4, 0])
    return ((adc[1] & 3) << 8) + adc[2]

print("센서 값 읽기 시작 (Ctrl+C 종료)")
print("-" * 40)

try:
    while True:
        mq2_ao   = read_channel(0)
        flame_ao = read_channel(1)
        print(f"MQ-2: {mq2_ao:4d} | 불꽃: {flame_ao:4d}")
        time.sleep(1)
except KeyboardInterrupt:
    print("\n종료")
finally:
    spi.close()
```

### 기대 출력
```
MQ-2:   48 | 불꽃:  312   ← 정상
MQ-2:  250 | 불꽃:  315   ← MQ-2 전조 감지
MQ-2:  870 | 불꽃:   10   ← MQ-2 + 불꽃 동시 감지
```

## 5. 아날로그 기반 AI 이상 감지 설계

### 핵심 개념

| 단계 | DO 값 | AO 값 | AI 판정 |
|------|-------|-------|---------|
| 정상 | 1 | 40~60 | 정상 |
| 전조 | 1 | 100~400 | **AI 이상** ← 핵심 |
| 감지 | 0 | 500~ | 경고 + AI 이상 |

DO가 아직 1(정상)인데 AO값이 평소보다 올라가면 AI가 먼저 경고.
이것이 단순 if문과 차별화되는 AI의 실제 역할.

### AI 피처 구성 (변경)

| 피처 | 설명 |
|------|------|
| mq2_ao | MQ-2 현재 아날로그 값 (0~1023) |
| flame_ao | 불꽃 센서 현재 아날로그 값 (0~1023) |
| mq2_ao_mean_5m | 최근 5분 MQ-2 평균값 |
| flame_ao_mean_5m | 최근 5분 불꽃 평균값 |
| mq2_ao_max_30s | 최근 30초 MQ-2 최대값 |
| hour | 현재 시간대 (KST) |

## 6. 라즈베리파이 전송 코드 (sensor_test.py 전체 교체)

```python
import RPi.GPIO as GPIO
import spidev
import requests
import time
from datetime import datetime, timezone, timedelta

# ── 설정 ──────────────────────────────────────────
SERVER_URL = 'http://서버IP:5000/api/event'
MQ2_PIN    = 17
FLAME_PIN  = 27
KST        = timezone(timedelta(hours=9))

# ── GPIO 설정 ──────────────────────────────────────
GPIO.setmode(GPIO.BCM)
GPIO.setwarnings(False)
GPIO.setup(MQ2_PIN,   GPIO.IN)
GPIO.setup(FLAME_PIN, GPIO.IN)

# ── SPI (MCP3008) 설정 ────────────────────────────
spi = spidev.SpiDev()
spi.open(0, 0)
spi.max_speed_hz = 1350000

def read_channel(channel):
    adc = spi.xfer2([1, (8 + channel) << 4, 0])
    return ((adc[1] & 3) << 8) + adc[2]

def send(payload):
    try:
        r = requests.post(SERVER_URL, json=payload, timeout=5)
        print(f"[전송] {r.status_code} | {payload['mq2_status']} / {payload['flame_status']} | MQ2_AO={payload['mq2_ao']} FLAME_AO={payload['flame_ao']}")
    except Exception as e:
        print(f"[전송 실패] {e}")

print("센서 모니터링 시작 (Ctrl+C 종료)")
print("-" * 60)

try:
    while True:
        # DO 값 (0=감지, 1=정상)
        mq2_raw   = GPIO.input(MQ2_PIN)
        flame_raw = GPIO.input(FLAME_PIN)
        mq2_status   = '감지됨' if mq2_raw   == 0 else '정상'
        flame_status = '감지됨' if flame_raw == 0 else '정상'

        # AO 값 (0~1023)
        mq2_ao   = read_channel(0)
        flame_ao = read_channel(1)

        payload = {
            'ts':           datetime.now(KST).isoformat(),
            'mq2_raw':      mq2_raw,
            'mq2_status':   mq2_status,
            'flame_raw':    flame_raw,
            'flame_status': flame_status,
            'mq2_ao':       mq2_ao,
            'flame_ao':     flame_ao,
        }
        send(payload)
        time.sleep(2)

except KeyboardInterrupt:
    print('\n종료')
finally:
    GPIO.cleanup()
    spi.close()
```

### AI 피처 추출 변경

```python
def extract_features(conn, current_ts: str, mq2_ao: int, flame_ao: int) -> list[float]:
    rows_30s = conn.execute('''
        SELECT mq2_ao, flame_ao FROM readings
        WHERE received_at >= datetime(?, '-30 seconds')
    ''', (current_ts,)).fetchall()

    rows_5m = conn.execute('''
        SELECT mq2_ao, flame_ao FROM readings
        WHERE received_at >= datetime(?, '-5 minutes')
    ''', (current_ts,)).fetchall()

    mq2_mean_5m   = sum(r['mq2_ao']   for r in rows_5m)   / max(len(rows_5m),   1)
    flame_mean_5m = sum(r['flame_ao']  for r in rows_5m)   / max(len(rows_5m),   1)
    mq2_max_30s   = max((r['mq2_ao']   for r in rows_30s), default=0)

    hour = datetime.now(KST).hour

    return [mq2_ao, flame_ao, mq2_mean_5m, flame_mean_5m, mq2_max_30s, hour]
```

### 학습 로직 변경

정상 상태(DO=정상)의 AO값만으로 학습 → 정상 AO 범위 학습 → 범위 벗어나면 이상 판정

```python
rows = conn.execute(
    "SELECT mq2_ao, flame_ao, received_at FROM readings "
    "WHERE mq2_status='정상' AND flame_status='정상' "
    "ORDER BY id DESC LIMIT 1000"
).fetchall()
```

## 9. SPI 활성화 트러블슈팅

### 문제
`dtparam=spi=on` 설정 후 재부팅해도 `/dev/spi*` 장치 파일이 생성되지 않음.

### 원인
`/boot/config.txt`에 `dtoverlay=tft35a:rotate=90` (TFT 디스플레이 오버레이)가 SPI 버스를 점유하고 있어 충돌 발생.

### 해결
```bash
sudo nano /boot/config.txt
# dtoverlay=tft35a:rotate=90 → #dtoverlay=tft35a:rotate=90 으로 주석처리
sudo reboot
```

재부팅 후 확인:
```
/dev/spidev0.0  /dev/spidev0.1
```
