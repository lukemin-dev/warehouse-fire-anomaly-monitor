# 면접 대비 Q&A

## 30초 소개

라즈베리파이와 MQ-2, 불꽃 센서를 이용해 창고 내 화재 전조를 감지하는 시스템을 만들었습니다. 센서의 디지털 값만 사용하면 정상/감지 정도만 판단할 수 있어 MCP3008 ADC를 추가해 아날로그 값을 수집했습니다. Flask 서버에서 데이터를 저장하고 IsolationForest로 이상 징후를 판단하며, Expo 모바일 앱에서 실시간 상태와 그래프, 경고 이력을 확인할 수 있습니다.

## 1분 소개

이 프로젝트는 창고 화재나 이상 징후를 조기에 감지하기 위한 IoT 모니터링 시스템입니다. 라즈베리파이 4가 MQ-2 가스/연기 센서와 SEN040132 불꽃 센서의 값을 읽고, MCP3008을 통해 아날로그 출력까지 수집합니다. 수집한 데이터는 Flask 서버로 전송되어 SQLite에 저장되고, IsolationForest 모델이 정상 패턴에서 벗어난 값을 이상 징후로 판단합니다. 모바일 앱은 Expo/React Native로 구현했으며 대시보드, 실시간 그래프, 경고 이력, AI 모델 상태 화면을 제공합니다. 단순 임계값 감지에서 끝내지 않고, 아날로그 값의 변화로 전조를 감지하려고 한 점이 핵심입니다.

## 자주 나오는 질문

### 왜 AI를 사용했나요?

초기에는 센서의 DO 디지털 출력만 사용했는데, 이 값은 `0` 또는 `1`이라 단순 if문과 차이가 거의 없었습니다. 그래서 MCP3008 ADC로 AO 아날로그 값을 수집했고, 정상 상태의 값 분포에서 벗어나는 전조를 감지하기 위해 IsolationForest를 사용했습니다.

### 왜 IsolationForest를 선택했나요?

실제 화재 데이터는 안전상 많이 수집하기 어렵고, 반대로 정상 데이터는 비교적 쉽게 모을 수 있습니다. IsolationForest는 라벨이 충분하지 않은 상황에서 정상 패턴과 다른 이상치를 탐지하는 데 적합해서 선택했습니다.

### 가장 어려웠던 문제는 무엇이었나요?

라즈베리파이에서 SPI가 활성화되지 않는 문제가 있었습니다. 원인을 확인해보니 TFT 디스플레이 오버레이가 SPI 버스를 점유하고 있었고, `/boot/config.txt`에서 해당 오버레이를 비활성화한 뒤 `/dev/spidev0.0` 장치가 정상적으로 생성되는 것을 확인했습니다.

### 단순 임계값과 어떤 차이가 있나요?

단순 임계값은 특정 값 이상 또는 이하일 때만 경고합니다. 이 프로젝트에서는 AO 값을 지속적으로 수집하고 정상 패턴을 학습해, DO가 아직 감지 상태로 바뀌기 전에도 평소와 다른 변화를 이상 징후로 볼 수 있게 했습니다.

### 프로젝트의 한계는 무엇인가요?

실제 화재 환경 데이터를 많이 확보하지 못했고, 센서값은 거리, 공기 흐름, 전원 안정성, 온습도에 영향을 받을 수 있습니다. 또한 현재 관리 API에는 인증이 없어서 운영 환경에서는 인증과 권한 관리가 필요합니다.

### 운영 수준으로 개선한다면 무엇을 하겠나요?

관리자 API 인증, 푸시 알림, 장기 데이터 저장소, 센서 전압 보호 회로, 모델 평가 지표를 추가하겠습니다. 특히 라즈베리파이 입력은 3.3V 기준이므로 MQ-2 AO 출력 보호를 회로적으로 보완하는 것이 중요합니다.

### 본인의 역할은 무엇인가요?

센서 배선과 라즈베리파이 수집 코드, Flask 서버 API, SQLite 저장 구조, IsolationForest 기반 이상 탐지 로직, Expo 모바일 앱 연동까지 전체 데이터 흐름을 구현했습니다.

## STAR 답변 예시

상황: 디지털 출력만으로는 AI 이상 감지의 의미가 약했습니다.

과제: 단순 감지가 아니라 전조를 감지할 수 있는 구조가 필요했습니다.

행동: MCP3008 ADC를 추가해 센서의 아날로그 값을 수집하고, 정상 값 기반 IsolationForest 모델을 서버에 적용했습니다.

결과: DO 값이 바뀌기 전에도 AO 값의 변화를 그래프로 확인하고, 이상 징후로 기록할 수 있는 구조를 만들었습니다.
