# 시연 시나리오

이 문서는 발표나 면접에서 프로젝트 동작 흐름을 설명하기 위한 시연 순서입니다.

## 1. 정상 상태

- 라즈베리파이에서 `sensor_test.py` 실행
- MQ-2와 불꽃 센서의 DO/AO 값을 2초마다 읽음
- Flask 서버의 `/api/event`로 JSON 데이터 전송
- 모바일 앱 대시보드에서 `정상 상태` 표시

확인 포인트:

- 네트워크 상태가 연결됨으로 표시되는지 확인
- MQ-2 AO 값과 불꽃 AO 값이 그래프에 누적되는지 확인
- 경고 이력에 새 경고가 생기지 않는지 확인

## 2. 가스/연기 전조 테스트

- MQ-2 센서 근처에 라이터 가스 또는 알코올을 짧게 근접
- DO가 아직 정상이어도 AO 값이 평소보다 상승할 수 있음
- 서버가 정상 패턴에서 벗어난 값을 이상 징후로 판단

확인 포인트:

- 그래프에서 MQ-2 AO 값이 상승하는지 확인
- 앱 경고 탭에 AI 이상 또는 가스 관련 경고가 기록되는지 확인
- 서버 `/api/alerts` 응답에 경고 이력이 추가되는지 확인

## 3. 불꽃 감지 테스트

- 불꽃 센서 방향에 작은 불꽃을 짧게 노출
- 불꽃 센서는 값이 낮아질수록 감지 가능성이 높음
- 서버가 불꽃 임계값 이하 값을 경고로 기록

확인 포인트:

- 대시보드의 불꽃 센서 상태가 이상 감지로 바뀌는지 확인
- 경고 이력에 불꽃 관련 기록이 남는지 확인

## 4. 앱 확인

- 대시보드: 센서 상태와 시스템 상태
- 그래프: AO 실시간 값, 이동평균, AI 이상치
- 경고: 경고 발생 시간과 센서값
- 설정: AI 모델 상태, 학습 데이터, 재학습/초기화 기능

## 발표용 요약

정상 상태에서 센서값을 수집하고, 가스/불꽃 테스트로 AO 값이 변할 때 서버가 이상 여부를 판단합니다. 모바일 앱은 서버 API를 주기적으로 조회해 상태와 그래프, 경고 이력을 보여줍니다.
